# Master of Epic 物理ダメージ計算webツール

**Master of Epic** の物理攻撃ダメージを、武器・ステータス・装備・Buffを組み合わせて試算するブラウザ用ツールです。

GitHub Pages でそのまま使えます。

https://sincostanrap-droid.github.io/Master-of-Epic-web-damage-tool/

> 非公式ツールです。ゲーム内仕様や各種データは更新・検証中のため、最終確認はゲーム内や公式/各種検証情報と照合してください。

---

## 主な機能

- 物理攻撃ダメージの試算
- 武器性能、攻撃力、命中、ディレイ、クリティカル率などの入力
- 種族・ステータス・スキル値を考慮した計算
- 装備登録と装備セット比較
- 装備カタログからの装備追加
- 装備BuffのON/OFFと効果編集
- 装備Buffによる物理与ダメージ、クリティカル、攻撃力変換などの反映
- Buffの競合グループ設定
- 条件に合う装備構成の最適化
- 計算結果の確認・比較・共有用表示

## 使い方

### Web版を使う

以下のURLを開くだけで使えます。

https://sincostanrap-droid.github.io/Master-of-Epic-web-damage-tool/

ページ更新後に表示が古い場合は、ブラウザで `Ctrl + F5` を押して再読み込みしてください。

### ローカルで使う

リポジトリを取得したあと、簡易サーバーで起動してください。

```bash
python -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000/
```

`index.html` を直接開いても動く場合がありますが、ブラウザの制限で一部のデータ読み込みが失敗することがあります。ローカル確認には簡易サーバーの利用を推奨します。

## 画面の概要

### ダメージ計算

武器性能、攻撃力、補正、Buffなどを入力して、物理攻撃ダメージを試算します。

### 装備登録

手持ちの装備やカタログから追加した装備を登録します。  
装備本体の追加ステータスと、装備Buffの効果を個別に編集できます。

### 装備カタログ

公式DB等から生成した装備データを検索し、装備登録へ追加できます。  
Buff名、追加ステータス、必要スキルなどの確認にも使えます。

### Buff

装備以外のBuffや、手動で追加したBuffを登録できます。  
競合グループを設定すると、同時に有効化できないBuffを整理しやすくなります。

### 最適化

登録済み装備・Buffから、指定条件に合う構成を探索します。  
ダメージ最大化だけでなく、命中や枠数などの条件確認にも使えます。

## 装備Buffについて

装備Buffは、装備に付いている常時発動系の効果を扱うための項目です。

現在は主に以下のような効果を扱います。

- 攻撃力%
- 攻撃力+
- 命中+
- クリティカル率+
- 物理与ダメージ+
- 魔力から攻撃力への変換
- 移動速度から攻撃力への変換
- スキル値+
- 計算未対応の表示用効果

カタログから装備を追加した場合、一部のBuff効果は自動入力されます。  
ただし、未検証・曖昧なものは表示用または手動確認用として残しています。

## データについて

このツールでは、複数のデータを組み合わせて使用しています。

- 装備カタログの生成データ
- 手動確認したBuff・効果ルール
- Wiki等の検証情報をもとにした併用/競合情報
- ツール内で編集したユーザー入力値

生成済みデータは主に以下に置かれています。

```text
src/data/generated/
```

手動管理データは主に以下に置かれています。

```text
data/manual/
```

開発・検証用の詳細メモは `docs/` を参照してください。

## 注意点

- このツールは非公式です。
- ゲーム仕様の変更により、計算結果が実際と異なる場合があります。
- 装備Buffや追加効果の一部は、まだ確認中です。
- 魔法ダメージや属性ダメージは、物理ダメージ計算とは別扱いです。
- 「計算未対応・表示用効果」は、表示や確認用であり、基本的に物理ダメージ計算へは反映されません。

## 開発メモ

通常利用ではビルド作業は不要です。  
HTML/CSS/JavaScriptの静的ファイルとして動作します。

データ更新や検証用スクリプトは `tools/` にあります。  
生成データを更新する場合は、対象スクリプトと `docs/` のメモを確認してください。
